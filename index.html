Docker Commands Reference Guide for Modern Development Workflows

📦
Container Management

docker run -d -t container_id
Run a container in detached mode with a pseudo-TTY
Launches container in background with terminal access
docker run --rm image_name
Automatically remove the container when it stops
Perfect for temporary containers and testing
docker run -v $HOSTDIR:$DOCKERDIR
Map a host directory to a container directory
Enables persistent data storage and file sharing
docker rm -v container_name
Remove a container and its associated volumes
Clean removal including all mounted volumes
docker start container_name
Start a stopped container
Resume a previously created container
docker stop container_name
Gracefully stop a running container
Sends SIGTERM then SIGKILL after timeout
docker restart container_name
Restart a container (stop then start)
Useful for applying configuration changes
docker pause container_name
Pause all processes in a running container
Temporarily freeze container execution

⚙️
Container Operations

docker unpause container_name
Resume all processes in a paused container
Restore normal container execution
docker kill container_name
Send a SIGKILL signal to terminate container
Force immediate container termination
docker attach container_name
Attach to a running container's STDOUT/STDERR
Connect to container's main process output
docker exec -it container_name bash
Execute an interactive command in running container
Open interactive bash shell in container
docker commit container_name new_image
Create a new image from container's changes
Save container state as reusable image
docker logs -f container_name
Display and follow container logs in real-time
Monitor container output continuously
docker inspect container_name
Display detailed container configuration
View complete container metadata and settings
docker ps -a
List all containers (running and stopped)
Complete container inventory with status

🖼️
Image Management

docker build -t myapp:1.0 .
Build an image from Dockerfile with tag
Create versioned image from current directory
docker images
List all locally stored Docker images
View image repository, tags, and sizes
docker rmi alpine:3.4
Remove specified image from local storage
Free up disk space by removing unused images
docker pull nginx:latest
Download image from Docker registry
Fetch latest version from Docker Hub
docker push myapp:1.0
Upload image to configured registry
Share image with team or deploy to production
docker tag myapp:1.0 myapp:latest
Create alias tag for existing image
Mark current version as latest release

🔧
Docker Compose

docker-compose up -d
Start multi-container application in background
Launch entire application stack from YAML definition
docker-compose down
Stop and remove all services and networks
Clean shutdown of complete application stack
docker-compose build --no-cache
Rebuild all services from scratch
Force fresh build ignoring Docker cache
docker-compose logs -f service_name
Stream logs from specific service
Monitor real-time output from individual service

💾
Volume & Network Management

docker volume create mydata
Create a named volume for persistent storage
Establish reusable data storage location
docker volume ls
List all available Docker volumes
View all persistent storage volumes
docker network create --driver bridge mynet
Create custom bridge network
Establish isolated communication network
docker network ls
Display all Docker networks
View available network configurations

🖥️
System Commands

docker system prune -a
Remove all unused containers, networks, and images
Comprehensive cleanup to reclaim disk space
docker info
Display comprehensive Docker system information
View daemon status, storage driver, and statistics
docker version
Show Docker client and server version details
Verify Docker installation and compatibility
docker events --since 1h
Monitor Docker events from last hour
Track container lifecycle and system events
